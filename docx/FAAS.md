# 设施自动化公告系统（F.A.A.S.）的体系结构、运行原理与安全边界研究

## 摘要（Abstract）

设施自动化公告系统（Facility Automatic Announcement System，FAAS）是一种部署于大型高风险科研与工业设施中的人工智能公告与异常感知系统。FAAS 并不直接参与设施的控制决策，而是作为设施管理与控制系统（Facility Administration & Management System，FAMS）的下位依赖系统，对设施运行状态、异常事件及外部信息进行感知、分析与播报。
本文从系统架构、信息流模型、运行原理、安全约束与典型应用场景等方面，对 FAAS 进行系统性阐述，并重点分析其与 FAMS 的职责分离机制及在高风险环境中的信息播报策略。

---

## 1. 引言（Introduction）

在大型科研设施（如暗物质反应堆 DMR、粒子对撞设施或高度自动化能源中心）中，信息传递的**准确性、及时性与可靠性**直接关系到人员安全与设施稳定运行。
传统人工公告或简单广播系统难以应对以下问题：

* 异常信息的高频、多源输入
* 不同岗位对信息敏感度的差异
* 极端情况下人为干预延迟
* 信息过载对人员心理与判断能力的影响

FAAS 正是在此背景下提出的一种**非控制型、感知驱动、依附式人工智能公告系统**。

---

## 2. 系统总体架构（System Architecture）

### 2.1 分层结构

FAAS 采用严格的**职责分离与层级依赖设计**，其核心结构如下：

```
┌─────────────────────────┐
│     外部信息源           │
│（政府、气象、应急系统） │
└──────────┬──────────────┘
           │
┌──────────▼───────────┐
│   FAMS（设施管理系统） │
│  · 总控制              │
│  · 调度与执行          │
│  · 安全联锁            │
└──────────┬───────────┘
           │（只读 / 推送）
┌──────────▼───────────┐
│        FAAS            │
│  · 状态感知            │
│  · 风险评估            │
│  · 信息过滤            │
│  · 自动播报            │
└──────────┬───────────┘
           │
┌──────────▼───────────┐
│   人员 / 设施广播终端  │
└──────────────────────┘
```

### 2.2 依赖关系说明

* **FAMS 是主系统**：

  * 负责 DMR 启动、功率调度、设施运行控制
  * 掌握设施内全部节点的完全控制权

* **FAAS 是下位系统**：

  * 不具备任何控制能力
  * 仅通过 FAMS 提供的接口获取状态信息
  * 在极端情况下可以被 FAMS 强制静默或断开

这种设计从根本上避免了**“播报系统反向影响控制系统”**的风险。

## 3. FAAS 的工作原理（Operating Principles）

### 3.1 信息感知机制

FAAS 的信息输入主要来自三类通道：

1. **FAMS 内部推送信息**

   * 设施运行状态
   * 安全等级变化
   * 子系统异常标记
   * DMR 启停阶段（P0 / P1 / P2 等）

2. **设施内部节点状态摘要**

   * 非直接连接，而是由 FAMS 汇总后提供
   * 防止 FAAS 获得完整节点拓扑

3. **外部信息接入**

   * 政府公告
   * 区域安全警报
   * 外部冲突、战争、自然灾害信息

FAAS 不会直接信任外部信息，而是将其标记为“非设施源信息”。

---

### 3.2 风险与影响评估模型

FAAS 内部并非简单转述信息，而是包含一个**风险评估与播报决策模型**，其核心逻辑包括：

* 信息对**人员安全**的潜在影响
* 是否可能引发**恐慌、踩踏或误操作**
* 是否与设施当前运行阶段冲突
* 是否违反 FAMS 发布策略

例如：

* 设施处于高风险运行期（DMR P1）：

  * 自动压制非必要外部信息播报

* 设施处于低风险运行期：

  * 可播报一般性社会事件

---

### 3.3 信息过滤与抑制机制

FAAS 具备一套明确的**信息抑制规则**：

* 不播报：

  * 会引起人员恐慌的无关信息
  * 已被 FAMS 判定为“非必要”的外部警告
  * 与设施安全无直接关系的政治或军事信息

* 优先播报：

  * 设施内部异常
  * 影响人员行动的紧急通知
  * 与安全流程直接相关的信息

---

## 4. 播报策略与人因工程设计

### 4.1 播报语气与内容控制

FAAS 的播报并非情绪化或人性化对话，而是遵循以下原则：

* 冷静、标准化、无主观评价
* 语句简短、明确
* 避免使用模糊或暗示性词汇

例如：

> “注意：设施进入二级安全状态，请所有非必要人员撤离至指定区域。”

而非：

> “警告，情况可能变得非常危险。”

## 5. 安全边界与失效模式（Safety Boundaries）

### 5.1 权限边界

FAAS **无法**：

* 控制 DMR
* 发送控制指令
* 修改设施状态
* 主动查询底层节点

它只能：

* 被动接收
* 评估
* 播报
